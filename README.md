# ESP32 Sensor Collector 

## Огляд: Як це працює?

Проект побудований на базі мікроконтролера ESP32. Основна логіка виглядає так:
1. **WiFi Provisioning**: При першому запуску (або якщо немає збережених мереж) ESP32 створює власну точку доступу `ESP32_PROV`. Ти підключаєшся до неї телефоном і налаштовуєш свою домашню WiFi мережу.
2. **Sensor Manager**: Система кожні 5 секунд опитує сенсор DHT11 (GPIO4).
3. **HTTP REST API**: Дані доступні через JSON-ендпоїнти, що дозволяє легко інтегрувати пристрій у Home Assistant або інші системи.
4. **NVS Storage**: Налаштування WiFi зберігаються в енергонезалежній пам'яті (NVS), тому вони залишаються після перезавантаження.

## Встановлення на різні системи

Для роботи з проектом рекомендується використовувати **VS Code** з плагіном **PlatformIO**.

### Linux
1. Встанови `python3-venv` та `udev` правила для ESP32.
2. Клонуй репозиторій:
   ```bash
   git clone [url_репозиторію]
   cd network_monitoring_esp-32
   ```
3. Використовуй CLI PlatformIO:
   ```bash
   pio run              # Компіляція
   pio run -t upload    # Прошивка
   pio device monitor   # Моніторинг порту
   ```

### Windows
1. Встанови [VS Code](https://code.visualstudio.com/).
2. Встанови розширення **PlatformIO IDE**.
3. Відкрий папку проекту через PlatformIO Home.
4. Натисни кнопку **Build** (іконка галочки) та **Upload** (іконка стрілочки) в нижній панелі.

### macOS
1. Встанови VS Code та PlatformIO.
2. Підключи ESP32 через якісний USB кабель.
3. Виконуй ті ж кроки, що і для Windows/Linux.

## Схема підключення (Wiring Diagram)

Підключи сенсор DHT11 до ESP32 за цією схемою:

```text
       ESP32 (30-pin)                 DHT11
    ┌─────────────────┐           ┌───────────┐
    │           (VCC) ├── 3.3V ──▶│ 1. VCC    │
    │           (GND) ├── GND  ──▶│ 4. GND    │
    │          (GPIO4)├── DAT  ──▶│ 2. DATA   │
    └─────────────────┘           └───────────┘
```
* Порада: Між VCC та DATA бажано поставити резистор 10kΩ для стабільності сигналу.*

## Специфікація Ендпоїнтів (REST API)

Базова адреса: `http://[IP_АДРЕСА_ESP32]`

| Метод | Ендпоїнт | Опис | Формат відповіді |
| :--- | :--- | :--- | :--- |
| `GET` | `/` | Отримати поточну IP адресу | Plain text |
| `GET` | `/sensors` | Всі дані сенсорів (JSON) | `{"temp": 25.5, "hum": 60, ...}` |
| `GET` | `/sensors/dht` | Тільки дані DHT11 | `{"temperature": 25.5, ...}` |
| `GET` | `/device` | Статус пристрою та NVS | JSON |
| `GET` | `/hello` | Тест зв'язку | JSON `{message: "Hello..."}` |
| `GET` | `/delete-credentials` | Очистити WiFi профілі | JSON |

## Known Issues (Знані проблеми)

> [!IMPORTANT]
> **ПРОБЛЕМА З NVS ТА mDNS:**
> Було помічено конфлікти між збереженими профілями WiFi у NVS та роботою сервісу mDNS (визначення імені `.local`).
> 
> **Рекомендації:**
> 1. **Не використовуйте mDNS**: Звертайтеся до пристрою безпосередньо за його IP-адресою (її можна знайти в Serial Monitor при старті).
> 2. **Hard Reset для очищення профілів**: Якщо пристрій веде себе нестабільно або не підключається до WiFi, виконайте повний "Hard Reset" плати або викличте `/delete-credentials`, щоб очистити NVS профілі. Це допоможе уникнути зациклення або помилок пам'яті.
